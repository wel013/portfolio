Project 3: Scheme Metacircular Evaluator

Overview

This project involved crafting an interpreter for a subset of the R5RS Scheme programming language entirely in Scheme itself. The primary objectives were to deepen proficiency in functional programming, including recursion, higher-order functions, functional data abstraction, and message passing. Additionally, the project aimed to elucidate the principles of interpretation and distinguish between static and dynamic scope in programming languages.

Key Components

  • Phase 1: Dictionaries and Environments: The initial phase centered on constructing abstract data types for dictionaries and environments using higher-order functions. The focus was on message passing and dispatch functions, utilizing Scheme symbols as messages.
  • Phase 2: Primitives and Evaluation: Implemented fundamental aspects of the Scheme interpreter, enabling the evaluation of basic Scheme expressions. This phase empowered the interpreter to handle simple expressions involving calls to primitive procedures and compound expressions composed of these foundational elements.
  • Phase 3: Special Forms: Extended the evaluation procedure in the interpreter to accommodate special forms. These forms differ from procedures by expecting unevaluated arguments, treating them as data instead of directly evaluating them.
  • Phase 4: Dynamic Scope: Introduced an extension to the Scheme language by incorporating a "mu" form that generates procedures utilizing dynamic scope instead of static scope. Dynamic scope with shallow binding was emphasized, altering the parent environment of newly created frames for procedure calls.
  • Developed an exhaustive set of test cases to rigorously evaluate the functionality and correctness of the interpreter. These tests encompassed various scenarios, ensuring the interpreter handled different expressions and special forms accurately.

Achievements

  • Successfully crafted a Scheme interpreter, showcasing adeptness in functional programming concepts and message passing.
  • Implemented complex evaluation procedures for special forms, enabling the interpreter to handle unevaluated arguments.
  • Illustrated the value of a metacircular evaluator by extending the Scheme language with dynamic scope

Technologies & Skills:

Scheme, Functional Programming, language interpretation principles, Static vs. Dynamic Scope.
