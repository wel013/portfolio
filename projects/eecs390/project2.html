Project 2: Scheme Lexer and Parser

Overview

This project entailed the implementation of a lexer and parser for the R5RS Scheme programming language. The primary objectives were to gain proficiency in functional programming paradigms and to deepen understanding of Scheme. It involved reasoning about lexical and syntactic structures of a language while creating a structured internal representation for Scheme computations.

Key Components

  • Phase 1 and 2: Lexer Development: Initially, separate functions were designed to lex different token types in Scheme. Each function targeted a specific token type (e.g., strings, booleans, characters, numbers, identifiers, punctuators) and handled error-catching mechanisms. In Phase 2, the focus shifted to developing a unified function capable of lexing any supported Scheme token.
  • Phase 3: Parser Implementation: This phase revolved around translating tokens into structured Scheme-level representations for subsequent evaluation or data manipulation. The parser, utilizing the read-token function, parsed tokens from standard input and generated Scheme data representing input expressions. It involved handling both simple expressions and compound expressions through recursive parsing.
  • Testing Phase: The final step involved devising comprehensive test cases to ensure the accuracy and reliability of the lexer and parser functionalities. Test cases were designed to cover a range of scenarios, validating the correctness of both phases of the implementation.

Achievements

  • Successfully implemented lexer functions for various token types, ensuring accuracy and error handling.
  • Designed a unified lexer capable of handling diverse Scheme tokens in a cohesive manner.
  • Developed a robust parser translating tokens into structured Scheme-level representations, facilitating evaluation and manipulation.
