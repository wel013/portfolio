Project 3: Simulation of Euchre Card Game

Overview

Building an image resizing program utilizing a seam-carving algorithm.

Key Components

  • Card:
    • Developed an Abstract Data Type (ADT) for a playing card, representing the rank and suit of a card. The Card ADT included overloaded operators for performing comparisons between different card types, enhancing flexibility within the game.
  • Pack:
    • Implemented an ADT that represents a pack of playing cards. The default Pack constructor initialized a full pack with cards arranged from lowest to highest. Additionally, a secondary Pack constructor was designed to read a pack from stream input, allowing for increased versatility in pack initialization.
  • Player:
    • Implemented two player types: a Simple AI player and a Human-controlled player. Players were created using dynamically allocated memory, employing a factory function that returned a pointer to a Player with the correct dynamic type. Each player class is derived from the Player abstract base class and provided overrides for each player function. These functions, including making trumps, adding and discarding cards, and playing tricks. SImple AI player adhered to a rule-based approach. The Human Player interacted with human input.
  • Driver Interface:
    • Developed a driver interface enabling users to run the Euchre game. This interface served as the entry point for executing the game simulation, providing an intuitive way for users to engage with and observe the game's progression.

Achievements

  • Established a robust Card ADT, enhancing game flexibility through overloaded operators for card type comparisons.
  • Implemented an efficient Pack ADT, offering methods for pack initialization from both default and stream input.
  • Created dynamic and engaging AI and Human players, showcasing proficiency in Derived Classes, Inheritance, and Polymorphism.

Technologies & Skills:

C++, Object-Oriented Programming (OOP), Abstract Data Types, Derived Classes, Inheritance, Polymorphism.
