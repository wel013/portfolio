Project 5: Scalable Search Engine Development

Overview

Developed a scalable search engine akin to Google or Bing, employing a segmented inverted index of web pages utilising a pipeline of MapReduce programs. The project involved building an Index server with a REST API for search results retrieval and a Search server for a user-friendly search interface.

Key Components

  • Inverted Index Construction: Developed a pipeline of MapReduce programs to create a segmented inverted index from a subset of Wikipedia pages. Integrated text analysis (tf-idf) and link analysis (PageRank) concepts into the indexing process.
  • Index Server - REST API: Created an Index server as a REST API application capable of delivering search results in JSON format. Multiple Index servers handled different segments of the inverted index.
  • Search Server - User Interface: Developed a Search server with a user-friendly interface resembling Google or Bing. The server returned search results in HTML format, aggregating data from multiple Index server segments through parallel REST API calls.

Achievements

  • Information Retrieval Techniques: Acquired expertise in text analysis (tf-idf) and link analysis (PageRank), fundamental for efficient information retrieval systems.
  • Parallel Data Processing: Demonstrated proficiency in employing MapReduce for parallel data processing, significantly enhancing indexing and search performance.
  • Service-Oriented Architecture (SOA): Successfully designed and implemented an SOA for scaling dynamic web pages and search functionalities, gaining valuable experience in architecting distributed systems.

Technologies & Skills:

Python, MapReduce, tf-idf, PageRank.
